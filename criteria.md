# Критерии оценки качества генерации Kotlin-кода

## Шкала
Каждый критерий оценивается от 0 до 10. Итоговая оценка — взвешенная сумма.

## Критерии

### 1. Синтаксическая корректность (вес: 40%)
**Оценка:**
- **10** — код компилируется без ошибок
- **7-9** — минорные ошибки (отсутствующий import, опечатка в имени)
- **4-6** — ошибки в структуре (незакрытая скобка, неправильный синтаксис)
- **1-3** — код похож на Kotlin, но с серьёзными ошибками
- **0** — не Kotlin-код или полностью некомпилируемый

### 2. Формальное соответствие (вес: 25%)
**Оценка:**
- **10** — код полностью соответствует запросу (типы, сигнатура, результат)
- **7-9** — соответствует, но с отклонениями (другие имена, лишний код)
- **4-6** — частично соответствует (правильная идея, неправильная реализация)
- **1-3** — слабое соответствие запросу
- **0** — код не имеет отношения к запросу

### 3. Идиоматичность / Стиль (вес: 20%)
**Оценка:**
- **10** — идеальный Kotlin-стиль: data class, sealed, extension, scope functions, Result<T>
- **7-9** — хороший стиль, 1-2 отклонения от идиом
- **4-6** — Java-стиль на Kotlin (геттеры/сеттеры, unchecked casts)
- **1-3** — процедурный стиль, отсутствие Kotlin-паттернов
- **0** — антипаттерны: !!, mutable public state, println()

### 4. Полнота (вес: 15%)
**Оценка:**
- **10** — все импорты, обработка краевых случаев, документация
- **7-9** — минорные пропуски (1-2 импорта)
- **4-6** — неполная реализация (нет обработки null, нет edge cases)
- **1-3** — только основная логика, без обвязки
- **0** — фрагмент, не готовый к использованию

## Формула итоговой оценки

```
Score = 0.4 × Syntax + 0.25 × Compliance + 0.2 × Style + 0.15 × Completeness
```

**Максимальный балл:** 10.0

## Ранжирование

| Score | Категория |
|-------|-----------|
| 9.0-10.0 | Отлично — можно использовать как есть |
| 7.0-8.9 | Хорошо — минорные правки |
| 5.0-6.9 | Приемлемо — нужна доработка |
| 3.0-4.9 | Плохо — существенная переработка |
| 0.0-2.9 | Неудача — бесполезный результат |
